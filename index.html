Module Assignment - Part 1: Theory Questions

V8 Engine

1. What is the V8 Engine?

The V8 Engine is Google's open-source JavaScript engine. It was developed for Google Chrome and is also used by Node.js. It converts JavaScript code into machine code so that the computer can execute it quickly and efficiently.

2. Why is the V8 Engine important for Node.js?

The V8 Engine allows Node.js to run JavaScript outside the browser. It compiles JavaScript directly into machine code, making Node.js applications fast and efficient.

3. What is the main responsibility of the V8 Engine?

The main responsibility of the V8 Engine is to execute JavaScript code. It compiles JavaScript into machine code to improve performance and execute programs efficiently.

Execution

4. What is the difference between Synchronous and Asynchronous execution?

Synchronous Execution:

  • Executes one task at a time.
  • Each task waits for the previous task to finish.
  • It blocks the execution of the next task.

Asynchronous Execution:

  • Does not wait for a task to finish.
  • Other tasks continue running while the asynchronous task is completed.
  • It makes applications faster and more responsive.

5. What is the Call Stack?

The Call Stack is a data structure that keeps track of function execution in JavaScript. When a function is called, it is pushed onto the stack, and when it finishes execution, it is removed from the stack.

Event Loop

6. What is the Event Loop?

The Event Loop is a mechanism in Node.js that continuously checks whether the Call Stack is empty. If the Call Stack is empty, it moves completed asynchronous tasks from the Callback Queue to the Call Stack for execution.

7. Why is the Event Loop needed in Node.js?

Node.js is single-threaded, so it can execute only one task on the Call Stack at a time. The Event Loop helps Node.js handle asynchronous operations efficiently without blocking the main thread.

8. How does the Event Loop help execute asynchronous code?

When an asynchronous operation such as setTimeout() or file reading starts, Node.js sends it to libuv. Once the operation finishes, its callback is placed in the Callback Queue. When the Call Stack becomes empty, the Event Loop moves the callback to the Call Stack, where it is executed.

libuv

9. What is libuv?

libuv is a C library used by Node.js that provides asynchronous I/O operations. It manages the Event Loop, timers, thread pool, file system operations, and networking.

10. What types of tasks are handled by libuv?

libuv handles many asynchronous tasks, including:

  • File System operations
  • Network operations
  • DNS lookups
  • Timers (setTimeout and setInterval)
  • Thread Pool tasks
  • Event Loop management
